# PRD（產品需求文件）- 線上算命系統

## 1. 專案概述

### 背景與動機
現代人生活步調快速、壓力繁重，常常在面臨學業、事業、感情或健康抉擇時，需要尋求心靈寄託與方向指引。然而，前往實體的傳統廟宇或算命館受限於距離與時間。為了滿足現代人隨時隨地的需求，我們希望打造一個「線上算命系統」，讓使用者能透過網路隨時抽籤、占卜，快速得到心靈上的平靜與建議。

### 目標用戶
- **年輕上班族與學生**：對面對未來抉擇感到迷惘，需要指引或第二意見的人。
- **有信仰及命理興趣者**：平時就有拜拜、抽籤或占卜習慣，但受限時間無法親自前往實體場所的民眾。

### 核心價值主張
打破地理與時間的限制，提供隨時隨地、便利、有趣且具備互動性的線上命理體驗。

---

## 2. 功能需求

以下為系統的主要功能清單：
1. **線上算命、占卜與抽籤**：提供多種模式（如媽祖靈籤、塔羅牌等）。
2. **儲存算命結果**：讓登入會員可以保留並回顧過去的抽籤與算命紀錄。
3. **線上捐獻香油錢**：提供模擬的小額捐款機制，讓使用者表達心意。
4. **會員註冊與登入**：支援帳號密碼註冊，管理個人檔案。
5. **社群分享功能**：允許把有趣的算命或抽籤結果分享給朋友。

### 使用者故事 (User Stories)
- 作為 **一般訪客**，我希望 **能線上進行簡單的抽籤或占卜**，以便 **快速獲得心靈指引與建議**。
- 作為 **註冊會員**，我希望 **能儲存過去的算命結果與籤詩**，以便 **日後隨時回顧與驗證**。
- 作為 **信眾 / 使用者**，我希望 **能在線上模擬捐贈香油錢**，以便 **表達對神明的敬意與還願**。
- 作為 **一般訪客**，我希望 **能順利註冊成為會員**，以便 **體驗進階功能（如儲存紀錄）**。
- 作為 **使用者**，我希望 **能一鍵將算命或抽籤結果分享到社群平台**，以便 **與朋友分享互動**。

---

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Python 的 **Flask** 框架。
- **前端渲染**：使用 **Jinja2** 模板系統結合現代化 CSS/JS。
- **資料庫**：使用 **SQLite** 作為主要資料庫（便於開發階段展示與部署）。

### 效能與安全考量
- **效能考量**：抽籤與查詢的動作應迅速回應（少於 1 秒），且在節慶高峰（如新年）需確保系統不易崩潰。
- **安全考量**：
  - 會員密碼須以雜湊（如 BCrypt）加密儲存。
  - 防範常見的網路攻擊（防堵 SQL Injection 與 XSS）。
  - 若香油錢金流為真實串接，需嚴格依循第三方金流的安全規範（本專案初版可設為模擬機制）。
